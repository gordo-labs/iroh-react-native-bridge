# iroh-react-native-bridge

React Native bridge for opening minimal [Iroh](https://www.iroh.computer/) peer
connections from iOS and Android apps.

The published package is:

```bash
npm install music-hub-iroh-bridge
```

This repository is independent from Iroh/n0. It is maintained by Gordo Labs for
Music Hub and shared as open source so other React Native teams can test,
review, and help harden the mobile Iroh path.

The React Native binding layer is built with
[jhugman/uniffi-bindgen-react-native](https://github.com/jhugman/uniffi-bindgen-react-native),
the maintained UniFFI-to-TypeScript/TurboModule generator used by this package.

## Status

Alpha. The bridge is usable for controlled device testing, but the API and
native packaging may still change before a stable 1.0 release.

What works today:

- Rust crate using `iroh` 1.x.
- React Native TurboModule package for iOS and Android.
- iOS `xcframework` and Android `.so` artifacts are included in the npm package.
- Minimal endpoint lifecycle: `start`, `stop`, `isRunning`, `nodeId`.
- Outbound QUIC connection to a remote Iroh endpoint using an advertised address
  hint.
- Length-prefixed binary frame send/receive for an app-level tunnel.
- Android JNI context setup for Iroh DNS/network support.
- Local tests for Rust lifecycle/framing and Music Hub tunnel integration.

Known limits:

- Not a full Iroh SDK for React Native.
- No public example app is included yet.
- Incoming mobile server mode is not exposed.
- The current JS API is callback/poll based and intentionally small.
- Real-device NAT/relay behavior still needs more external reports.
- The package is developed against modern React Native/TurboModule builds.

See [docs/STATUS.md](./docs/STATUS.md) for the detailed support matrix.

## API Preview

```ts
import { getIrohBridge } from 'music-hub-iroh-bridge';

const bridge = getIrohBridge();
if (!bridge) {
  throw new Error('Iroh bridge package unavailable');
}

await bridge.start();
const localNodeId = await bridge.nodeId();

const connection = await bridge.connect(remoteNodeId, addressHint);
connection.onMessage((bytes) => {
  // bytes is a Uint8Array containing one app-level frame.
});

await connection.send(new Uint8Array([1, 2, 3]));
await connection.close();
await bridge.stop();
```

`addressHint` is required for mobile dialing today. It can be a direct address
hint generated by a desktop/service peer, or a JSON object containing the remote
node id and usable transport addresses. A display-only relay URL is not enough.

## Contributing

External support is welcome. The most useful contributions right now are:

- Device reports with OS version, React Native version, and network conditions.
- Reproduction cases for failed dialing, native linking, or relay/NAT behavior.
- PRs that improve packaging, diagnostics, CI, or examples without widening the
  API surface unnecessarily.
- Reviews from people familiar with Iroh, UniFFI, React Native TurboModules,
  CocoaPods, Gradle, and mobile networking.

Before opening a PR, read [CONTRIBUTING.md](./CONTRIBUTING.md). Use the GitHub
issue templates so maintainers have enough information to reproduce platform
bugs.

## Security

This package opens encrypted Iroh sessions, but the bridge does not decide your
application's authorization model. Applications must authenticate peers and
messages at their own protocol layer.

Do not report security issues in public GitHub issues. See
[SECURITY.md](./SECURITY.md).
