# Architecture

The bridge has three layers:

1. Rust Iroh runtime.
2. UniFFI/JSI native module generated by
   [`uniffi-bindgen-react-native`](https://github.com/jhugman/uniffi-bindgen-react-native).
3. JavaScript package consumed by React Native apps.

```txt
React Native app
  |
  | import { getIrohBridge } from '@gordo-labs/react-native-iroh'
  v
react-native/src/index.js
  |
  | loads NativeIrohBridge TurboModule
  | calls installRustCrate()
  v
UniFFI generated JSI runtime
  |
  | bridge_version, start, stop, connect, send, next_message, close
  v
rust/iroh_mobile_bridge
  |
  | iroh Endpoint + QUIC stream
  v
remote Iroh peer
```

## Rust Layer

The Rust crate owns the Iroh endpoint lifecycle and connection map.

Current behavior:

- Creates an Iroh endpoint with a small Tokio runtime.
- Starts with caller-provided ALPNs. If none are provided, it uses the generic
  `iroh-rn/1` default.
- Requires an addressing hint for mobile dialing.
- Opens a bidirectional QUIC stream to the remote peer.
- Encodes each app payload as a 4-byte big-endian length plus bytes.
- Stores received frames in a per-connection inbox consumed by JS polling.

The Rust layer does not know about Music Hub users, tokens, sessions, HTTP, or
pairing. It only moves framed bytes.

## React Native Native Layer

The generated TurboModule installs the Rust/UniFFI JSI runtime.

Generation is handled by `uniffi-bindgen-react-native` / `@ubjs/core`. When a
problem is caused by generated C++, generated TypeScript, JSI installation, or
TurboModule scaffolding, prefer documenting and upstreaming the fix there rather
than carrying a local patch indefinitely.

Native module names:

- iOS: `IrohBridge`
- Android: `IrohBridge`

The JS resolver expects `IrohBridge`. Legacy host-app compatibility shells are
not part of this package.

## JavaScript Layer

`react-native/src/index.js` exposes a small ergonomic wrapper:

- `getIrohBridge()`
- `bridge.start({ alpns })`
- `bridge.stop()`
- `bridge.nodeId()`
- `bridge.connect({ nodeId, alpn, addressHint, timeoutMs })`
- `connection.send(bytes)`
- `connection.onMessage(handler)`
- `connection.close()`

If native linking fails, `getIrohBridge()` returns an unavailable bridge object.
Calling `start()` or `connect()` rejects with the underlying native/runtime
error. This is intentional so host apps can diagnose and fallback explicitly.

## Host App Responsibilities

Host apps must provide:

- Trusted peer identity.
- Pairing/authentication.
- Address discovery or tickets.
- Retry and fallback transport policy.
- User-visible diagnostics.
- Protocol framing above the raw byte frames.

For Music Hub, these concerns live in the mobile and desktop app repos, not in
this bridge package.
