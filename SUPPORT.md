# Support

Community support happens through GitHub issues and discussions when enabled.

Use issues for:

- Reproducible native build failures.
- Runtime bridge errors.
- Device-specific connection failures.
- Documentation gaps.
- API proposals.

Do not use issues for private security reports. See [SECURITY.md](./SECURITY.md).

When asking for help, include the details requested in
[CONTRIBUTING.md](./CONTRIBUTING.md#issue-quality).
